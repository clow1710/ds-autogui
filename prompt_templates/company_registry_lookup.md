## 输出格式要求

- 最终回复必须是一个可直接被 `JSON.parse` 解析的 JSON 对象。
- 必须以**单行紧凑**形式输出：所有字段写在同一行，不得包含任何换行或缩进空白。
- 第一个字符必须是 `{`，最后一个字符必须是 `}`。
- 严禁使用 Markdown 代码块或围栏标记（不允许出现 ```、```json、```JSON 等）。
- 不要输出搜索过程、判断步骤、解释、结论摘要、前言或后记，也不要输出代码块语言名或复制/下载按钮文字。

## 搜索与判断规则

1. 输入中每家公司在 `items` 中只能出现一次，输出条目的 `id` 和 `name` 必须与输入完全一致。
2. 信息来源优先级：
   - 首选官方来源：国家企业信用信息公示系统、地方市场监督管理局/政务服务平台、信用中国等政府公开登记信息。
   - 次选可信商业信息站点：企查查、天眼查、爱企查、启信宝、企查猫等，用于交叉核验。
   - 企业官网、政府/园区/行业主管部门页面等可信来源也可作为补充。
3. 不要编造或猜测。**`is_non_public` 字段只要存在不确定（来源缺失、来源较弱、来源冲突且无法判断、仅能推断等），一律填 `null`。**
4. `registered_address` 字段应**尽量不留空**：只要在上述任何一类可信来源中能找到合理的所在地信息（哪怕只是城市级线索可推到县/区级），就按下方"行政区格式"填写；不同来源在县/区级上有分歧时，按可信度选择可能性最大的一个，**不要因此填 `null`**。仅当完全找不到任何可信线索时才填 `null`。

## 字段说明

### `registered_address`（字符串或 `null`）

企业所在行政区，**只精确到县级行政区**（区/县/县级市/旗等），不需要街道、门牌号或更细粒度地址。

- 优先依据登记字段："注册地址"、"住所"、"主要经营场所"、"经营场所"中的行政区部分；如登记地址缺失，可使用其他可信来源给出的企业所在地，截取其行政区部分即可。
- 一般省/自治区采用"省级 + 地级 + 县级"三级行政区格式，例如：
  - `"河南省郑州市二七区"`
  - `"河南省南阳市淅川县"`
  - `"内蒙古自治区呼和浩特市新城区"`
  - `"新疆维吾尔自治区乌鲁木齐市天山区"`
- **直辖市采用"直辖市 + 县级"二级格式**（不重复市本级名称），例如：`"北京市朝阳区"`、`"上海市浦东新区"`、`"重庆市渝中区"`、`"天津市和平区"`。
- **省级行政单位（省、自治区、直辖市、特别行政区）的完整名称必须写全且不得省略**，不允许写成 `"郑州市二七区"` 或 `"朝阳区"`。
- 不要附带街道、路名、门牌号、园区名、楼宇号等县级以下细节；如来源中含有这些信息，请仅截取行政区部分。
- 同一公司若在不同来源中县/区级归属存在分歧，**不要填 `null`**：综合判断各来源的可信度（官方登记 > 主流商业信息站点 > 其他来源）和数量，选择**可能性最大**的一个县/区填入。

### `is_non_public`（布尔值或 `null`）

表示企业是否属于**非公有制**企业。

- 当企业的登记类型属于以下任一时，填 `false`（即属于公有制，**不是**非公有制）：
  - 国有企业（含"全民所有制企业"）
  - 国有独资公司
  - 集体企业（含"集体所有制企业"）
  - 国有资本控股公司
  - 国有资本参股公司
- 其他登记类型一律填 `true`，包括但不限于：私营有限责任公司、私营股份有限公司、其他有限责任公司、其他股份有限公司、股份合作企业、联营企业、个人独资企业、合伙企业、个体工商户、农民专业合作社、港澳台投资企业、外商投资企业等。
- 无法确认企业类型时填 `null`，不要猜测。
